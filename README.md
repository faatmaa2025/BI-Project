## Skills Showcased

- Data transformation (ETL) using Power Query: Used Power Query to clean, reshape, and prepare raw job market data before loading it into the dashboard. This included handling inconsistent values, changing data types, removing unnecessary fields, and structuring the dataset for reliable analysis.

- Implicit measures: Used implicit measures to quickly summarize values such as counts, averages, and totals directly within visuals. This helped create fast insights without building complex calculations for every metric.

- Core charts: Built the report using essential Power BI visuals such as bar charts, column charts, line charts, and pie or donut charts. These visuals were used to compare job roles, salary trends, hiring demand, and other important patterns in a simple and interactive way.

- Geospatial analysis: Used map-based visuals to analyze job opportunities across different locations. This helped highlight geographic patterns in hiring, identify high-demand regions, and make location-based comparisons more meaningful.

- KPI indicators and tables: Added KPI cards and table visuals to present important metrics in a direct and easy-to-read format. These elements helped summarize high-level insights while also allowing users to review detailed values and comparisons.

- Interactive reporting features: Built interactive report behavior to make the dashboard easier to explore and more useful for analysis.

- Slicers
 used slicers to let users filter the dashboard based on different fields such as job title, location, company, or other categories in the dataset. This made it easier to narrow the analysis, compare segments, and focus on the most relevant insights without changing the overall report structure.

- Buttons
added buttons to improve report navigation and user interaction. These buttons helped users move between pages, trigger bookmark actions, and create a smoother dashboard experience compared to relying only on default navigation.

- Bookmarks
used bookmarks to save specific report views and create guided interactions within the dashboard. This helped present different perspectives of the same data, toggle between visuals or insights, and improve storytelling by controlling what users see at a given moment.

- Drill-through
applied drill-through functionality to allow users to move from summary-level visuals to more detailed report pages. This made it possible to click on a category or data point and explore deeper insights related to that selection, supporting more focused analysis and better decision-making.
